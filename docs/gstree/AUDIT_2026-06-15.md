## Critical Issues (Fix Before PyPI Publish)

### C1 ‚Äî Missing timeouts in `upgrade.py` subprocess calls
**File:** `src/gstree/upgrade.py:34, 45, 55`

All three `subprocess.run()` calls are network-dependent but have no `timeout`:
- `git pull` ‚Äî can hang indefinitely on slow/unreachable remote
- `uv tool upgrade` ‚Äî downloads from network
- `gstree --version` ‚Äî fast but no safeguard

**Fix:**
```python
subprocess.run([...], timeout=60, capture_output=True, text=True)
```

### C2 ‚Äî `parse_porcelain` accesses `line[0]` / `line[1]` without length check
**File:** `src/gstree/status.py:49-50`

Malformed git output (corrupted repo, git version mismatch) produces lines < 2 chars ‚Üí `IndexError`. Caught upstream by `_collect_safe` but fails at the wrong layer.

**Fix:**
```python
if len(line) < 2:
    continue
```

## High Priority

### H1 ‚Äî `collect_repo_status()` has zero tests
**File:** `src/gstree/status.py:13`

Most critical function (fetches all git metadata) has no unit or integration tests of its own. Only tested indirectly through CLI subprocess tests.

**Recommended:** integration test with a real temp repo covering: clean repo, detached HEAD (empty branch string), ahead/behind counts.

### H2 ‚Äî `--depth` accepts negative values or 0 silently
**File:** `src/gstree/cli.py:16`

`--depth -1` or `--depth 0` causes confusing behavior (no repos found) with no error message.

**Fix:**
```python
if args.depth < 1:
    sys.stderr.write("gstree: error: depth must be at least 1\n")
    return 1
```

### H3 ‚Äî Renderer internals (`_build_tree`, `_render_node`, `_format_repo_state`, `Palette.repo`, `Palette.dir`) have no tests
**File:** `src/gstree/renderer.py`

Tree-building and formatting logic is only exercised through CLI subprocess tests. Direct unit tests with `Palette(color=False)` would be fast and precise.

## Medium Priority

### M1 ‚Äî Inconsistent error reporting style across modules
- `cli.py` ‚Üí `sys.stderr.write(f"gstree: error: ...")`
- `upgrade.py` ‚Üí `print(..., file=sys.stderr)`
- `scanner.py` ‚Üí `print(f"gstree: warning: ...", file=sys.stderr)`

No single utility. Cosmetic but worth unifying before public release.

### M2 ‚Äî `_collect_safe` catches bare `Exception`
**File:** `src/gstree/scanner.py:53`

`except Exception` catches programmer errors, masking bugs. Should be narrowed:
```python
except (subprocess.CalledProcessError, subprocess.TimeoutExpired, OSError) as exc:
```

### M3 ‚Äî `_git_optional` timeout swallowed without context
**File:** `src/gstree/status.py:83`

When `_git_optional` times out, it returns `None` silently. The caller treats this as "not tracking" ‚Äî incorrect if the repo IS tracking but git was slow.
